+ 将浮点型转成整型 可能丢失数据精度（四舍五入浮点转整型）

+ 高类型转换成低类型int64转换成int32也可能引起精度丢失

+ 不同的整型或者不同的浮点型不能直接参与运算

+ 	在编译阶段操作系统会认为是不同类型，不能参与运算，go语言硬性规定。
    
    ```
    type int1 int
    type int2 int
    ```
    
+ 这段代码可以执行

  ```
  ch := 'A' //底层就是rune
  var n int32 = 100
  fmt.Println(ch*n) //可以计算
  fmt.Printf("%T",ch) //rune==int32
  ```
