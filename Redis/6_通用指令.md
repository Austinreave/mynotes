##### key 特征

+ key是一个字符串，通过key获取redis中保存的数据

**key 的重复问题**

+ key是由程序员定义的

+ redis在使用过程中，伴随着操作数据量的增加，会出现大量的数据以及对应的key

+ 数据不区分种类、类别混杂在一起，极易出现重复或冲突

+ redis为每个服务提供有16个数据库，编号从0到15

+ 每个数据库之间的数据相互独立

**db 基本操作**

+ 切换数据库：`select index`

+ 数据移动：`move key db`

+ 数据清除 `flushdb 清除一个库 `,`flushall 清除所有库`
